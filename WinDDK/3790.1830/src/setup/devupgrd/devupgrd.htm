Introduction

Devupgrd, the Microsoft General Device Driver Pack Migration DLL, allows system administrators to suppress hardware messages on user machines and to install drivers for devices not included with the standard Windows 2000 product. This allows devices to be installed during Setup in a manner that is transparent to the end user in the corporate environment.

This article introduces the use of Devupgrd to support migration to Windows 2000 from Windows 95/98.

Note: Devupgrd also provides a workaround for a migration issue that can occur in certain cases where the user has a device for which there is no built-in Windows 2000 support. While running the Upgrade Wizard under Windows 95/98, the user might click the Have Disk button to specify the location for Windows 2000 driver files to be installed as part of migration. A problem can occur if the user-supplied INF file for the device refers to one or more Microsoft-provided INF files that have the same names in Windows 95/98 and Windows 2000.

In such cases, the Include= entry in the user-supplied INF will invoke the specified Needs= section in the Windows 95/98 INF, rather than using the Windows 2000 INF file as was intended. This might result in the user being prompted for Windows 95/98 media, even though such files aren’t required for Windows 2000 installation. The solution for this problem is to use the Devupgrd Migration DLL as described in this article.

Devupgrd Migration DLL

Devupgrd provides a registry-based mechanism for identifying hard­ware and associated Windows 2000-compatible drivers that the Upgrade Wizard would otherwise consider incompatible. During migration, Devupgrd will:

§     Read device driver information from its registry location.

§     Suppress upgrade messages for associated devices.

§     Add the necessary information to the Setup information file to ensure that these devices are installed during device installation.

 

Devupgrd will be provided in future DDKs from Microsoft, and will be provided as a download on the web at:
http://www.microsoft.com/hwdev/driver/

Registering the Devupgrd Migration DLL

Before it can perform device migration, Devupgrd must be added to the Windows 95/98 registry so that the Windows 2000 Upgrade code will load and process it during system migration. To do this, Devupgrd should be listed in the Windows 95/98 registry in the following way:

1.       Add the following key to the Windows 95/98 registry:

HKLM\Software\Microsoft\Windows\CurrentVersion\Setup\Migration DLLs

2.       Add the following REG_SZ value to this key:

Microsoft Device Upgrade Pack = “<windows directory>\win9xmig\DevUpgrd"

 

Where <windows directory>\win9xmig\DevUpgrd  is the full path to the subdirectory containing the Devupgrd files under the Windows 95/98 directory, as described in the following section.

 

Important: Microsoft recommends that vendors include this registry information in new Windows 95/98 and Windows Millennium Edition (Windows Me) installations, in anticipation of future upgrades to Windows 2000 and later versions. For information and tools for adding registry entries under Windows 95/98/Me, see the Microsoft Windows 98 Resource Kit.

Registering Devices for the Devupgrd Migration DLL

Only one instance of the Devupgrd Migration DLL is required on a machine. When Devupgrd runs, it will handle all device drivers listed in its registry section.

To add a device driver to the registry so that it will be correctly detected and migrated by Devupgrd, add an entry to the registry in the following form:

1.       Add the following key to the Windows 95/98 registry:

HKLM\Software\Microsoft\Windows\CurrentVersion\Setup\UpgradeDrivers

 

2.       Add the following REG_SZ value to this key:

<Plug and Play ID> = “<windows directory>\win9xmig\DevUpgrd\FooW2K\Foo.INF”

 

Where:

§     <Plug and Play ID> is the Plug and Play ID of the hardware device to be handled.

§     <windows directory>\win9xmig\DevUpgrd\FooW2K\Foo.INF is the full path to the Windows 2000 installation INF for that device, “Foo” is replaced with your vendor name, and “Foo.INF” is replaced with the name of the Windows 2000 INF file. This path must refer to a local drive on the machine which will be accessible during the GUI-mode setup portion of the upgrade to Windows 2000.

Devupgrd will use the directory of the Windows 2000 installa­tion INF as the base install directory. All files required to install the device must be in this directory or one of its subdirectories.

If the INF file is omitted from the REG_SZ value, as in <windows directory>\win9xmig\DevUpgrd\FooW2K, Devupgrd will load the first INF file it finds in this subdirectory. Devupgrd cannot differentiate between multiple INF files, so there must be only one INF file in this subdirectory. For clarity, it is strongly recommended that the REG_SZ value contain the full path and INF file name, as in <windows directory>\win9xmig\DevUpgrd\FooW2K\Foo.INF.

Insert the following lines into your INF file to install the Migration DLL plus a copy of your driver package on the target system:

[DestinationDirs]

DevUpgrd-Files = 10, Win9xmig\DevUpgrd

W2KDriver-Files = 10, Win9xmig\DevUpgrd\FOOW2K 
    ; Replace "FOO" with your vendor name

 

[DDInstall]

AddReg = DevUpgrd-AddReg

CopyFiles = DevUpgrd-Files, W2KDriver-Files

 

[DevUpgrd-AddReg]

HKLM,"Software\Microsoft\Windows\CurrentVersion\Setup\Migration DLLs","Microsoft Device Upgrade Pack",,%10%\Win9xmig\DevUpgrd

HKLM,"Software\Microsoft\Windows\CurrentVersion\Setup\UpgradeDrivers","<Plug and Play ID>",,%10%\Win9xmig\DevUpgrd\FOOW2K\Foo.INF

 

[DevUpgrd-Files]

Migrate.dll

 

[W2KDriver-Files]

Foo.inf ; Name of your INF goes here

Foo.sys ; Name of your driver file(s) goes here

 

Note for testers: If the solution is run on a checked build of Windows 2000, the developer will see the following error message:

"Cannot append - load \<name of INF file>.inf [Error = 2]"

 

This message can be clicked through and ignored.

Call to Action for Devupgrd Migration DLL

§     To ensure correct installation behavior with vendor-supplied INF files where one or more Include= entries refer to an INF file with the same name as a file provided with Windows 95/98Me use the Devupgrd Migration DLL as described in this article.

System administrators are also strongly encouraged to use this method to support automated rollout of upgrades for Windows 95/98 to Windows 2000.

§     The Devupgrd Migration DLL, when available, will be provided for download at:
http://www.microsoft.com/hwdev/driver/

§     For more information about installation of device drivers, see the Windows 2000 driver development kit (DDK):
http://www.microsoft.com/ddk/W2kDDK.htm
